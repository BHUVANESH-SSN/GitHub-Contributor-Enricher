## Methodology

1. Contributor data was collected from the GitHub API.
2. Contributors were initially classified as internal or external using public GitHub metadata.
3. LinkedIn URLs were resolved using an Apify search workflow.
4. LinkedIn profile data was enriched from local cache and profile scraping results to infer employer and tenure.
5. Employer values were bucketed to the target set: `OpenAI`, `Google`, `Anthropic`, `xAI`, or `Unknown`.

## Notes and Assumptions

- The process is automated and avoids manual profile lookups.
- Rows with incomplete enrichment are retained with `Unknown`, empty URLs, or lower-confidence values where needed.
- Internal vs external status may be refined after enrichment when employment data provides a stronger signal than GitHub metadata.
